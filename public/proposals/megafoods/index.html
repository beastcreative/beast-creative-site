01 · The Urgent Finding

Right now, Google can't see either of your websites

Both of your sites are effectively absent from Google today — for two different, fixable reasons:

  • megaalimentos.com (Mexico): built on WordPress with a single "discourage search engines" setting left switched on after development. It's actively telling Google to stay away. A one-click fix.
  • mega-foods.com (U.S., newly launched): isn't blocking anything, but because of how it was rebuilt last month, its pages arrive empty to search engines (content only loads inside a visitor's browser), and it's missing the basic files engines look for. Google arrives and finds nothing to list.
Why this matters more than followers

When a shopper — or a wholesale buyer — asks Google or ChatGPT "best chamoy brand" or "chamoy distributor near me," the answer is assembled from content the engines can actually read. Today, none of yours qualifies. Competitors like Tajín fill that space by default.

The good news

The Mexico site is a one-click fix we're happy to hand you, because it's the right thing to do. The U.S. site needs a bit more (its content has to be made visible to search engines), but it's well-defined work — not a rebuild from scratch. Fixing discovery only opens the door. The opportunity is in what you build behind it.

B · Developer Punch-List

The exact fixes, by priority

Priority: P0 do first / blocking   P1 high impact   P2 cleanup

mega-foods.com · React (Vite) SPA on Firebase → goal: static rendering (SSG) from CDN

Switch from client-side rendering to SSG P0

Served HTML body is just <div id="root"></div>; crawlers that don't run JS see an empty page.

Fix: Pre-build static HTML for every route at build time and deploy to Firebase Hosting. Add an SSG step to the Vite build (vite-react-ssg) or migrate to Next.js static export. Foundation for everything below.

Add a real robots.txt & XML sitemap P0

Both /robots.txt and /sitemap.xml return the app shell.

Fix: Generate a static sitemap.xml listing every route and a robots.txt that allows crawling and references it.

Fix canonical & identity domain P0

No canonical exists; og:url/og:image point to mega-foodsusa.web.app.

Fix: Add a self-referencing canonical on https://mega-foods.com/, update OG/Twitter tags to production, and 301-redirect the .web.app domain.

<!-- on every page -->
<link rel="canonical" href="https://mega-foods.com/products/" />
<meta property="og:url" content="https://mega-foods.com/products/" />

Unique title & meta description per page P1

Every page serves the identical title and one generic description.

Fix: Set unique, keyword-relevant head tags per route, emitted into static HTML at build time.

Return real 404 status codes P1

Unknown URLs return 200 OK with the app shell.

Fix: Configure hosting/SSG so unknown routes return a proper 404 with a real not-found page.

Add structured data (schema) P1

No Schema.org markup anywhere.

Fix: JSON-LD for Organization, Product, and Recipe. Recipe schema is the biggest win for rich results and AI citation.

Wire up social links P1

No real social URLs in the site code.

Fix: Add live profile URLs to the footer/header and to the Organization schema sameAs array.

Internationalization (ES / EN) + performance P2

Languages toggle in JS on one URL; single 731 KB bundle; ~2.3s TTFB.

Fix: Serve each language on its own path (/en/, /es/) with reciprocal hreflang; SSG + CDN + route-based code-splitting solves the speed.

megaalimentos.com · WordPress — settings + SEO plugin (Yoast/RankMath)

Turn off "Discourage search engines" P0

Every page carries noindex,nofollow.

Fix: WordPress admin → Settings → Reading → uncheck "Discourage search engines…" → Save. One click. Then request re-indexing in Search Console.

Fix dead placeholder links P1

The "Ir a formulario" CTA and the phone number both point to href="#".

Fix: Point the CTA at the real form, and make the phone a tel: link.

<!-- after --> <a href="tel:+528180482600">+52 81 8048 2600</a>

Titles, meta descriptions & H1 P1

Generic title, no descriptions, zero H1s.

Fix: With an SEO plugin, set unique keyword titles/descriptions per page and add one descriptive <h1> each.

Sitemap + structured data P1

/sitemap.xml doesn't resolve; 0 schema blocks.

Fix: Once indexing is on, Yoast/RankMath generates the sitemap; add Organization, Product & Recipe JSON-LD.

Caching, bloat, alt text & legacy P2

Caching disabled; 17 CSS + 34 JS + 6 font requests; 15/34 images missing alt; IE8 shims.

Fix: Enable caching, combine/minify assets, consolidate fonts, add alt text, remove dead IE8 scripts.

Do-first shortlist (the P0s)

  1. megaalimentos.com: uncheck "Discourage search engines" (1 click).
  2. mega-foods.com: switch to static rendering (SSG).
  3. mega-foods.com: add robots.txt + sitemap.xml.
  4. mega-foods.com: add canonicals and fix the staging-domain identity tags.
